# چک‌لیست تست — کلاب VIP کلینیک لمون

این چک‌لیست تمام حالت‌های اصلی رندر و auth را پوشش می‌دهد. قبل از هر انتشار (Publish) یک بار از بالا تا پایین اجرا کن.

---

## 1. کاربر لاگین‌نشده (Anonymous)

مرورگر تازه / حالت ناشناس / بعد از Sign Out.

- [ ] صفحه `/` باز می‌شود بدون خطا
- [ ] انیمیشن intro (کارت‌های مزایا) **نمایش داده نمی‌شود**
- [ ] بخش "EXCLUSIVE INVITATION" + عنوان «عضویت در کلاب VIP» دیده می‌شود
- [ ] دکمه «ورود با گوگل» قابل کلیک است
- [ ] فرم اصلی (انتخاب مزایا، تماس، آدرس، عکس) **نمایش داده نمی‌شود**
- [ ] رفتن دستی به `/admin` → ریدایرکت به `/auth`
- [ ] رفتن دستی به `/auth` → فرم لاگین ایمیل/رمز نشان داده می‌شود
- [ ] کنسول مرورگر بدون خطای قرمز

## 2. کاربر عادی جدید (هیچ submission ندارد)

لاگین با گوگل/ایمیل که قبلاً فرم را پر نکرده.

- [ ] بعد از لاگین، مستقیم به مرحله ۰ (انتخاب مزایا) می‌رود
- [ ] انیمیشن intro کارت‌ها یک بار اجرا می‌شود
- [ ] progress bar روی `0%` و step `مزایا`
- [ ] انتخاب مزایا → دکمه «ادامه» فعال می‌شود
- [ ] طی مراحل تماس → آدرس → عکس → تایید بدون خطا
- [ ] آپلود عکس کار می‌کند و quality check نشان داده می‌شود
- [ ] ثبت نهایی → صفحه success با ref_code + وضعیت «در حال بررسی»
- [ ] رفرش صفحه → مستقیم به step 4 (success) می‌رود (نه از اول)
- [ ] تگ‌های مزایا در صفحه success خوانا هستند (متن سفید روی پس‌زمینه طلایی پررنگ)

همان حساب کاربر عادی بعد از یک بار ثبت فرم.

- [ ] باز کردن `/` → بدون نمایش فرم، مستقیم به step 4 (وضعیت)
- [ ] ref_code، شماره، آدرس و مزایای انتخاب‌شده درست نمایش داده می‌شوند
- [ ] وضعیت فعلی (new / contacted / approved / rejected) با رنگ مناسب
- [ ] کاربر **نمی‌تواند** فرم جدیدی ثبت کند
- [ ] رفتن به `/admin` → پیام "دسترسی غیرمجاز" + ایمیل نمایش داده می‌شود

## 5. تست‌های متقاطع (Edge Cases)

- [ ] لاگین → بستن تب → باز کردن مجدد → سشن حفظ شده
- [ ] دو تب همزمان: Sign Out در یک تب → تب دیگر هم خارج می‌شود
- [ ] تغییر اندازه ویوپورت موبایل (393px) → چیدمان به‌هم نمی‌ریزد
- [ ] تغییر اندازه ویوپورت دسکتاپ (≥1024px) → چیدمان درست
- [ ] RTL: تمام متون فارسی راست‌چین، آیکون‌ها در جای درست
- [ ] رفتن به یک URL غیرموجود → صفحه 404 (نه خطای خام)
- [ ] قطع موقت اینترنت موقع submit → پیام خطای toast، state از دست نمی‌رود
