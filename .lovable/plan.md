## هدف

بلافاصله بعد از لاگین (شماره یا گوگل)، یک مرحله ساده و لوکس برای دریافت **نام و نام خانوادگی** اضافه شود — قبل از انتخاب مزایا. اسم در دیتابیس همراه فرم ذخیره می‌شود و در صفحه تأیید نمایش داده می‌شود.

## تجربه کاربری

```
لاگین → [جدید: نام و نام خانوادگی] → مزایا → تماس → آدرس → تصویر → تایید
```

- یک کارت glass با یک input تکی که هم نام و هم فامیل را در یک خط می‌گیرد.
- placeholder: «نام و نام خانوادگی»
- اعتبارسنجی: حداقل دو کلمه (نام + فامیل)، هر کدام ≥ ۲ حرف.
- دکمه «ادامه» با همان استایل btn-gold موجود.
- اگر کاربر قبلاً ثبت کرده باشد (برگشت دوباره)، نامش از دیتابیس بارگذاری می‌شود و این مرحله رد می‌شود.

## تغییرات فنی

1. **دیتابیس** — افزودن ستون `full_name text` به جدول `vip_submissions` (nullable برای سازگاری با رکوردهای قبلی، با CHECK روی طول حداقل ۳ در سطح اپ).
2. **state جدید** در `src/routes/index.tsx`:
   - `const [fullName, setFullName] = useState("")`
3. **آرایه steps** بازتعریف:
   - `["نام", "مزایا", "تماس", "آدرس", "تصویر", "تایید"]` (۶ مرحله، تأیید = step 5)
4. **منطق مرحله‌ها**:
   - افزودن یک `StepShell` جدید برای `step === 0` (نام)، و شیفت همه step های فعلی به جلو (مزایا → 1، تماس → 2، آدرس → 3، تصویر → 4، تایید → 5).
   - به‌روزرسانی `canNext`، `next/back`، و همه مقایسه‌های step.
5. **بارگذاری از DB** (`loadForUser`):
   - select شامل `full_name` شود؛ اگر موجود بود `setFullName(data.full_name)` و step را روی ۵ بگذارد.
6. **submit** (`handleSubmit`): `full_name: fullName.trim()` در payload insert.
7. **صفحه تأیید** (`Confirmation` component): پراپ `fullName` اضافه شود و بالای ref code نمایش داده شود مثل: «{نام}، درخواست شما ثبت شد».
8. **اعتبارسنجی** برای مرحله نام:
   ```ts
   const nameOk = fullName.trim().split(/\s+/).filter(p => p.length >= 2).length >= 2;
   ```

## نکات حفظ سبک

- همان design tokens (`btn-gold`, `glass`, `gold-text`) و همان انیمیشن‌های StepShell.
- RTL کامل، input وسط‌چین، یک آیکون کوچک (مثلاً `User` از lucide) داخل input.
- بدون افزودن وابستگی جدید.
